# 에이전틱 아키텍처

에이전틱 아키텍처는 에이전트 인공 지능(AI) 프레임워크의 구조와 설계를 의미합니다.

에이전틱 아키텍처는 가상 공간과 워크플로 구조를 형성해서 에이전틱 AI 시스템 안에 있는 AI 모델을 자동화합니다.

## 에이전틱(Agentic) vs 비에이전틱(Non-Agentic) 아키텍처

| 구분       | 에이전틱 아키텍처                       | 비에이전틱 아키텍처               |
|----------|---------------------------------|--------------------------|
| 기본 개념    | 에이전트처럼 목표를 가지고 자율적으로 행동하는 AI 구조 | 입력 → 출력 중심의 단순 AI 구조     |
| 자율성      | 높음 (사람 개입 없이 일정 수준의 결정·행동 가능)   | 낮음 (사용자 입력에만 반응)         |
| 목표 지향성   | 명확한 목표를 설정하고 달성하려고 행동           | 명시적 목표 없음 또는 단일 요청 처리    |
| 의사결정 방식  | 계획, 추론, 평가를 반복하며 동적 결정          | 정적 추론, 단발성 응답            |
| 도구 사용    | 외부 도구(API, DB, 시스템)를 능동적으로 활용   | 일반적으로 도구 사용 없음 또는 제한적    |
| 오케스트레이션  | 필수 (워크플로, 상태 관리, 플래너 등)         | 불필요하거나 매우 단순             |
| 컨텍스트 관리  | 장·단기 메모리 활용 가능                  | 제한된 컨텍스트 윈도우             |
| 문제 해결 범위 | 복잡하고 다단계 문제에 적합                 | 단순·선형 문제에 적합             |
| 확장 형태    | 단일 에이전트 또는 다중 에이전트 협업 가능        | 단일 모델 중심                 |
| 대표 사용 사례 | IT 자동화, RAG+툴 기반 워크플로, 자율 시스템   | 텍스트 생성, 요약, 의미 분석, 기본 챗봇 |
| 시스템 복잡도  | 높음                              | 낮음                       |
| 개발 난이도   | 높음 (설계·운영 부담 큼)                 | 낮음 (빠른 적용 가능)            |

## 에이전틱 아키텍처의 유형

| 아키텍처 유형       | 구조                       | 주요 특징              | 강점                            | 약점                    | 우수 사용 사례                 |
|---------------|--------------------------|--------------------|-------------------------------|-----------------------|--------------------------|
| 단일 에이전트       | 하나의 에이전트가 모든 인식·결정·행동 수행 | 중앙 집중식, 독립 실행      | 설계·운영 단순, 예측 가능, 빠른 실행, 비용 효율 | 확장 한계, 복잡한 다단계 작업에 취약 | 단순 챗봇, 개인화 추천 시스템        |
| 다중 에이전트 (공통)  | 여러 전문 에이전트가 협력           | 역할 분담, 협업 기반 문제 해결 | 복잡한 문제 대응, 유연성, 전문성 활용        | 오케스트레이션 복잡, 통신 비용     | 복합 문제 해결, 전문 도메인 결합      |
| 수직 AI 아키텍처    | 리더 에이전트 + 하위 에이전트 계층     | 중앙 통제, 명확한 역할      | 작업 효율성, 책임 명확                 | 병목 현상, 단일 장애 지점       | 워크플로 자동화, 승인 프로세스, 문서 생성 |
| 수평 AI 아키텍처    | 동등한 에이전트 간 P2P 협업        | 분산 의사결정, 병렬 처리     | 창의성, 혁신, 병렬 작업                | 조정 어려움, 의사결정 지연       | 브레인스토밍, 복잡한 문제 해결        |
| 하이브리드 AI 아키텍처 | 수직 + 수평 혼합               | 동적 리더십, 협업적 통제     | 적응성, 유연성, 균형 잡힌 구조            | 설계·운영 복잡, 리소스 관리 어려움  | 전략 기획, 장기 프로젝트, 동적 워크플로  |
